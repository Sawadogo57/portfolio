# Politique de Sécurité (Security Policy)

## Versions prises en charge (Supported Versions)

En tant que projet de portfolio personnel, seule la version la plus récente déployée sur la branche `main` est activement maintenue.

| Version | Prise en charge |
| ------- | ------------------ |
| v1.0.x  | ✅ Oui             |
| < 1.0   | ❌ Non             |

## Signalement d'une vulnérabilité (Reporting a Vulnerability)

La sécurité de ce portfolio est une priorité, car elle reflète mes standards professionnels en tant qu'étudiant en Master Cybersécurité. Si vous découvrez une faille de sécurité (XSS, mauvaise configuration CSP, fuite de données, etc.), je vous prie de suivre les étapes suivantes :

1. **Ne pas ouvrir de "Issue" publique** sur GitHub pour signaler une faille.
2. Envoyez-moi un rapport détaillé par email à : **[TON-EMAIL@ICI.com]**.
3. Je m'engage à accuser réception du rapport sous 48 heures et à proposer un correctif dans les plus brefs délais.

### Ce que j'apprécie dans un rapport :
* Une description claire de la vulnérabilité.
* Les étapes pour reproduire la faille (Proof of Concept).
* Des recommandations de remédiation si possible.

## Engagement de non-poursuite
Si vous agissez de bonne foi et respectez cette politique de divulgation responsable, je m'engage à ne pas engager de poursuites judiciaires à votre encontre.
